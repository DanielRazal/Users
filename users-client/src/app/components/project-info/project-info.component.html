Users Project - Server

Server Overview

  • Login and registration system using RESTful API for CRUD operations.
  • Working with emails using SendGrid for sending and receiving emails.
  • Using Microsoft SQL Server database and DbContext for storing and managing user data.
  • Enabling CORS (Cross-Origin Resource Sharing) to allow communication between the server and the client.
  • Token authentication for securing API endpoints and user sessions.
  • Implementing server-side validations for data input verification and error handling.
  • Using DTOs (Data Transfer Objects) for data validation and to define the contract between the client and server.
  • Using environment and configuration files for managing application settings and deployment configurations.
  • Implementing interfaces for defining contracts and achieving loose coupling.
  • Using built-in attribute classes for applying validation rules, such as Required, StringLength, etc.
  • Utilizing the wwwroot folder to serve static files, such as HTML, CSS, JavaScript, and images.

Users Project - Client

Client Overview

  • Implementing authorization and authentication using AuthGuard in the client application with canActivate for route protection.
  • Using Angular's HttpClientModule for making HTTP requests from the Angular client to the API.
  • Using Swal (SweetAlert) library for displaying user-friendly alerts and notifications in the client application.
